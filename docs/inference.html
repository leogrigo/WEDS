TinyML Inference Pipeline

Technical documentation for the TinyML risk prediction models on the WEDS ESP32 Node, focusing on performance, safety, and stability.

Data Pipeline

The system processes raw sensor readings through a continuous pipeline. Raw sensor data (Temperature, Humidity, Pressure) is collected and validated to prevent NaN values. The data is then aggregated into a 7-day rolling history utilizing RTC-retained memory. Features are scaled using predefined means and standard deviations, and any subnormal (Denormal) floats are zeroed to prevent FPU stalling. The normalized data is then explicitly typed (Float32 or Int8) and packed into the 64KB Tensor Arena for the TFLite Interpreter.

Model Execution & Quantization Handling

The firmware dynamically supports both Float32 and Int8 (Quantized) models to prevent FPU hardware crashes and optimize performance. Before packing the tensor, the code explicitly checks input_->type. If it expects kTfLiteInt8, the normalized float values are scaled by input_->params.scale, shifted by input_->params.zero_point, and clamped to the [-128, 127] range. This dynamic type casting logic ensures that the hardware can execute the model safely regardless of whether it's unquantized or quantized, avoiding mismatches that could crash the Xtensa FPU.

Failsafes & FreeRTOS Stability

Running Machine Learning on constrained edge devices requires strict hardware safeguards:

  • ✔ 16-byte Memory Alignment: The Tensor Arena is strictly aligned to 16-byte boundaries (alignas(16)) as required by TFLite Micro's memory allocator. This prevents misaligned memory access that causes repeated stall cycles during SIMD instructions, which would otherwise starve the Task Watchdog Timer.
  • ✔ Task Stack Size: The Arduino loopTask is explicitly provisioned with over 24KB of stack space (WEDS_NODE_CYCLE_TASK_STACK_BYTES) via a custom override in main.cpp, preventing stack overflow during the deep call chain of interpreter_->Invoke().
  • ✔ Manual Watchdog Feeding: The FreeRTOS Task Watchdog Timer (TWDT) is manually reset (esp_task_wdt_reset()) right before model invocation and risk scoring to prevent the 30-second watchdog from firing and rebooting the MCU mid-inference.
