WEDS Firmware Flow & Loop Architecture

An architectural breakdown of the Wildfire Early Detection System's node execution path, featuring Hardware Deep Sleep, Dynamic Duty Cycling, and a Watchdog-Safe TinyML Inference Loop.

Firmware Loop Execution Flow

The ESP32 firmware utilizes a linear "Wake → Process → Sleep" one-shot pipeline optimized for true hardware deep sleep. Upon every boot or sleep wakeup, the entire cycle executes sequentially, and the MCU drops directly into a power-off state to maximize battery conservation.

RTC-Backed Persistence and Time Synchronization

Since deep sleep turns off all SRAM domains, standard volatile variables are lost on every cycle. To maintain a 7-day rolling average for TinyML time-series features (deltas and rolling averages) without external RTC chips or complex internet connections, the firmware leverages ESP32 RTC slow memory:

  • ✔ RTC Memory Reservation: Variables tagged with RTC_DATA_ATTR are mapped to the 8KB RTC memory segment that remains powered on during deep sleep. This includes the DailyBucket ring buffer structure.
  • ✔ Virtual Clock Synchronization: A virtual timestamp (rtc_virtual_timestamp) is retained in RTC memory. Before deep sleep, the processor computes the precise elapsed active time, adds the calculated sleep duration, and advances this virtual clock. This ensures the bucketing and time-series logic remains coherent across resets.

Step-by-Step Architecture

1

Wakeup & Initialization

ESP32 boots. Serial outputs and I2C buses (Wire/Wire1) are initialized. Peripherals are booted, and LoRa radios configure with parameters.

Boot / Setup
2

Sensor Read, NaN Check & Retry Loop

Sensing reads environmental conditions (BME688 or BMP280/AHT20 in hardware mode, or the Box-Muller simulation generator in simulation mode). A while(true) retry loop guards against transient NaN outputs from the simulator's Box-Muller transform. If temperature or humidity evaluate to NaN, the node logs a warning and retries locally. Only if the sensor system is persistently broken does the node fall back to a 5-second deep sleep.

Data Collection
3

EMA & TinyML Inference (Watchdog-Safe)

Averages and deltas are calculated. The 7-feature normalized vector is compiled into TFLite interpreter inputs. A Denormal Float Filter explicitly flushes any subnormal floats (FP_SUBNORMAL) to zero to prevent the Xtensa FPU from stalling. A Dynamic Type Check explicitly validates if the model expects kTfLiteFloat32 or kTfLiteInt8 before packing, aborting safely on mismatch. esp_task_wdt_reset() is called before inference to prevent the Task Watchdog Timer from expiring. The loopTask stack is correctly enforced to 24 KB via a getArduinoLoopTaskStackSize() override in main.cpp. TinyML then computes the fire risk score.

Edge Machine Learning
4

Network & RX Window

The LoRa receiver is kept polling for 250ms to intercept any incoming alert commands or configuration parameters from the gateway. Status is dispatched.

Communication
5

Deep Sleep Pipeline

Radio is shut down into standby. Dynamic duty cycle sets the sleep target. Virtual time is advanced. The MCU enters deep sleep, waking via internal timer.

TinyML Memory Requirements

Two hard constraints must be satisfied to prevent TG1WDT_SYS_RST watchdog resets during interpreter_->Invoke():

Constraint Value Location
tensor_arena_ alignment alignas(16) WedsRiskScore.h
loopTask stack size 24576 bytes (24 KB) platformio.ini · WedsNodeConfig.h
TWDT feed around Invoke() esp_task_wdt_reset() WedsRiskScore.cpp

The 16-byte arena alignment is required by TFLite Micro's internal bump allocator (kBufferAlignment = 16). Without it, the Xtensa LX7 SIMD load instructions (EE.VLD.128.IP) raise alignment exceptions inside quantized op kernels, accumulating enough stall cycles to trigger the 3-second TWDT timeout. The 24 KB stack prevents loopTask overflow during the ~18–22 KB Invoke() call chain depth.

Fail-Safe Sensor Fault Path

If readSensors() returns an invalid sample or any of temperature / humidity evaluate to NaN, the node follows a fail-safe retry path that avoids both the TinyML inference pipeline and the duty-cycle sleep scheduler:

  • 1. Enter a local retry loop — log [NODE_WARN] Sensor fault detected and call readSensors() again immediately.
  • 2. If the fault persists (e.g., hardware I2C error), call Serial.flush() and enter a 5-second deep sleep without advancing rtc_virtual_timestamp.
  • 3. MCU reboots and retries the full sensor read cycle — gateway blackout window is ≤ 5 s instead of 30 min. In simulation mode, the Box-Muller clamp ensures NaN is never produced, so this path is never reached.
